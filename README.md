# Social Media Sentiment Analysis & Business Insight Dashboard

MCA Major Project (23ONMCR-753) — built with Python, Streamlit, scikit-learn, and IBM Watson.

---

## 1. What's in this folder

```
sentiment_project/
│── app.py                 # Streamlit dashboard (frontend + backend, all in Python)
│── train_model.py         # Cleans data, trains ML model, saves it
│── watson_helper.py       # IBM Watson NLU integration
│── requirements.txt       # Python dependencies
│── data/
│   ├── sentiment140.csv    # <-- REPLACE this with the real Kaggle dataset (demo data included for testing)
│   ├── clean_sample.csv    # auto-generated by train_model.py, used for dashboard charts
│   ├── cluster_summary.csv # auto-generated: K-Means cluster themes
│   └── top_hashtags.csv    # auto-generated: trending hashtags
│── models/
│   ├── sentiment_model.pkl
│   ├── vectorizer.pkl
│   ├── kmeans_model.pkl      # Audience Segmentation
│   └── engagement_model.pkl  # Engagement Prediction (simulated target)
```

### Modules included

| # | Module | Technique | Notes |
|---|---|---|---|
| 1 | Sentiment Analysis | TF-IDF + Logistic Regression | Core module, trained on real labels |
| 2 | Audience/Content Segmentation | K-Means clustering (unsupervised) | Groups posts into themes from TF-IDF vectors |
| 3 | Trend/Hashtag Detection | Regex extraction + frequency count | Surfaces trending hashtags/keywords |
| 4 | Engagement Prediction | Linear Regression | Trained on a **simulated** engagement score (Sentiment140 has no real likes/shares) — disclose this in your report |
| 5 | Watson Comparison | IBM Watson NLU API | Side-by-side validation against your own model |
| 6 | Prediction History | SQLite (RDBMS) | Logs every live prediction; Create/Read/Delete demonstrated |

## 2. Get the real dataset (do this today)

1. Go to: https://www.kaggle.com/datasets/kazanova/sentiment140
2. Click **Download** (you'll need a free Kaggle account) — it's a ~240MB zip
3. Unzip it. You'll get a file like `training.1600000.processed.noemoticon.csv`
4. Rename it to `sentiment140.csv` and place it inside `data/`, replacing the demo file
5. In `train_model.py`, `SAMPLE_SIZE = 50000` is already set to keep training fast — raise
   it if you have time and want higher accuracy, but 50k is plenty for a good report.

---

## 3. Run it locally

```bash
pip install -r requirements.txt
python train_model.py        # trains + saves the model (takes a few minutes)
streamlit run app.py         # launches the dashboard in your browser
```

---

## 4. Set up IBM Watson (for the "Watson Analytics" subject tie-in)

1. Go to https://cloud.ibm.com/registration and create a free **Lite** account
2. In the IBM Cloud catalog, search **"Natural Language Understanding"** and create a
   Lite (free) instance
3. Open the service → **Manage** tab → copy your **API Key** and **URL**
4. Set them as environment variables before running the app:

   **Windows (PowerShell):**
   ```powershell
   $env:WATSON_APIKEY="your_api_key_here"
   $env:WATSON_URL="your_service_url_here"
   ```
   **Mac/Linux:**
   ```bash
   export WATSON_APIKEY="your_api_key_here"
   export WATSON_URL="your_service_url_here"
   ```
5. Go to the **"Watson Comparison"** page in the app and test it — you'll see your model's
   prediction next to Watson's, side by side.

If you run short on time, this piece is optional — the app will still run fine and just
show "Watson not available" on that page. Even without wiring it up, keep the
`watson_helper.py` file and a short write-up in your report describing how it *would*
integrate — examiners mainly want to see you understand the concept.

---

## 5. Deploy to the cloud (for the "Cloud" subject tie-in)

1. Create a free GitHub account if you don't have one, and push this whole folder to a
   new GitHub repository
2. Go to https://share.streamlit.io and sign in with GitHub
3. Click **"New app"**, select your repo, and set the main file to `app.py`
4. If you configured Watson, add `WATSON_APIKEY` and `WATSON_URL` under
   **Advanced settings → Secrets**
5. Click **Deploy** — you'll get a live public URL in ~2 minutes. Put this link and a
   screenshot in your report and demo it live in your viva.
